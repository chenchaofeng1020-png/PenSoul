# 实施计划 - 优化热点雷达板块

## 1. 后端开发 (`server/index.cjs`)

### A. 数据结构增强
- **图片与链接透传**：修改 `clusterTrends` 聚类函数，从原始热点数据中提取第一张可用的图片 (`image`) 和主链接 (`url`)，保存到聚类后的对象中。
- **36Kr 数据完善**：为 36Kr 的模拟数据添加示例图片链接，确保前端有图可示。

### B. 实现“自定义获取”接口
- **新增接口**：`POST /api/trends/fetch-custom`
- **参数支持**：`platform` (知乎/微博/36Kr), `category` (分类标签)
- **爬虫逻辑升级**：
  - **知乎**：支持根据分类映射到子榜单（如：科技数码 -> `billboard/digital`，体育 -> `billboard/sport`）。
  - **微博**：支持根据分类映射到子分类（如：娱乐 -> `cate=entrank`，社会 -> `cate=socialevent`）。
- **处理流程**：定向抓取 -> LLM 聚类 -> 合并入库 -> 返回新数据。

## 2. 前端开发 (`src/components/TrendRadar/TrendList.jsx`)

### A. 热点卡片 UI 升级
- **图片展示**：在卡片布局中增加图片展示区。如果有图显示缩略图，无图显示精美的默认占位图标。
- **直达链接**：在卡片右上角或底部添加“外部链接”图标按钮，点击即可在新标签页打开原始热点网页。

### B. “手动获取”功能入口
- **交互设计**：在列表头部添加“+”或“筛选获取”按钮。
- **操作流程**：
  1. 用户点击按钮，弹出简易对话框。
  2. 选择目标平台（如：微博）和感兴趣的分类（如：娱乐）。
  3. 点击“立即获取”，前端调用 `/api/trends/fetch-custom` 接口。
  4. 获取成功后自动刷新列表，展示新抓取的内容。

## 3. 验证计划
- 检查知乎和 36Kr 热点是否显示图片。
- 点击卡片上的链接图标，确认能否正确跳转。
- 测试“手动获取”功能，例如选择“微博+娱乐”，确认能否抓取到娱乐类热搜并显示在列表中。
