## 2. 背景与目标

### 2.1 背景
当前“内容规划”板块主要依赖人工录入计划。用户在已有完善的“产品规划”（画像、竞品、功能、卖点等）数据后，缺乏一个能将这些“静态信息”转化为“动态选题”的桥梁。
用户希望接入 AI，利用已有的产品规划数据作为上下文，自动发散出有创意、不呆板的选题建议，并支持一键转化为内容计划（Schedule）。

### 2.2 核心目标
1.  **自动化选题**：一键读取产品规划上下文，生成高质量内容选题。
2.  **发散性与创意**：**关键差异点** —— 既要“懂产品”，又不能“只说产品”。AI 需要结合营销思维、受众心理和平台特性进行发散，避免生成的选题像产品说明书一样生硬。不需要所有选题都必须要说到自己的产品。
3.  **无缝工作流**：生成的选题可直接转化为“新建计划”（Content Item），为后续的“AI 辅助撰写”打下基础。
4.  **选题要符合不同平台“号”的定位**：例如小红书的“号”定位是“分享知识、解决问题”，而抖音的“号”定位是“分享生活、引发互动”。因此，生成的选题要符合不同平台的定位，避免“只说产品”。还包括要符合这个号预定的人设和受众需求。需要有对应的产品方案来确保人设定位这个逻辑，例如由AI给出用户确认，或者用户自行输入等。

## 3. 用户场景 (User Stories)

### 3.1 基础场景
> **作为** 运营经理
> **我希望** 在制定下周公众号推文计划时，能让 AI 根据最近上线的“协作功能”和“项目经理”画像，给我 5 个既能体现产品价值，又能引发职场焦虑共鸣的选题
> **以便于** 我能快速筛选出 2 个好点子，填入内容日历。

### 3.2 创意发散场景 (重点)
> **作为** 市场专员
> **我希望** AI 不要只是复读功能点，而是能结合“行业趋势”或“生活场景”来以此类推
> **例如** 不是生成“如何使用我们的导出功能”，而是生成“为什么 90% 的加班都是因为报表做太慢？（附解决方案）”
> **以便于** 内容更具吸引力和点击率。

## 4. 功能详细设计

### 4.1 核心功能：AI 选题生成器

#### 4.1.1 入口与交互
- 在“内容规划”/“内容日历”视图上方增加显著按钮：✨ **AI 灵感选题**。
- 点击后右侧滑出“灵感配置面板”。

#### 4.1.2 配置参数 (Input)
用户可调整以下参数来控制 AI 的发散程度：
1.  **参考数据源 (多选)**：
    - [x] 产品核心定义 (定位/Slogan/价值主张)
    - [x] 目标画像 (Pain Points/Goals)
    - [ ] 竞品分析 (差异化打法)
    - [ ] 近期功能卡片 (Feature Updates)
    - [ ] 关键消息 (Messaging)
2.  **创意发散度 (Temperature)**：
    - **保守 (精准)**：严格围绕功能点，适合教程/更新公告。
    - **平衡 (推荐)**：结合场景与痛点，适合常规运营。
    - **开放 (脑洞)**：强关联行业话题、职场情绪、趣味段子，适合破圈传播。
3.  **目标平台与形式**：
    - 平台：公众号、小红书、Twitter、LinkedIn、抖音脚本等。
    - 形式：图文、短视频脚本、长文章、快讯。
4.  **自定义关键词 (补充)**：
    - 允许用户输入当前热点或特定主题（如“双十一”、“年终总结”、“AI 替代人工”），强迫 AI 将产品与该词结合。

#### 4.1.3 生成逻辑 (Processing)
后端调用火山引擎接口，构建 Prompt 链：
1.  **Context Extraction**: 提取选中的产品规划数据。
2.  **Role Play**: 设定 AI 为“拥有 10 年经验的增长黑客”或“爆款文案策划”。
3.  **Creativity Injection**:
    - 强制要求 AI 使用“类比”、“反直觉”、“情绪共鸣”、“数据佐证”等手法。
    - **反呆板机制**：Prompt 中明确禁止使用“XX产品介绍”、“XX功能上线”等平铺直叙的标题，除非用户选择“保守”模式。
4.  **Format Output**: 返回 JSON 格式的选题列表。

#### 4.1.4 结果展示与操作 (Output)
- **卡片式列表**：每张卡片包含：
    - **标题** (爆款风格)
    - **核心角度** (Angle，如“利用恐惧心理”、“强调效率提升”)
    - **推荐平台**
    - **预期效果** (预估互动率/难度)
    - **简要大纲** (3-4 个 bullet points)
- **操作**：
    - **换一批**：对不满意的结果进行重刷。
    - **微调**：点击编辑标题或大纲。
    - **采纳 (加入计划)**：点击后，自动调用 `createContentItem` 接口，将标题、平台、大纲（存入 body 或备注）、建议时间写入数据库，并在日历中显示。

## 5. 补充的增值功能点 (Value-Add Features)
*以下是您可能未考虑到，但能显著提升产品体验的功能点：*

### 5.1 "系列化" 选题模式 (Campaign Mode)
- **痛点**：单一选题容易碎片化，缺乏连贯性。
- **功能**：AI 不只生成单条，而是生成一个**“一周/一月运营战役”**。
- **示例**：针对“新版本发布”，生成：
    - Day 1 (预热): 悬念海报文案（小红书）
    - Day 3 (发布): 核心功能深度以此类推（公众号）
    - Day 5 (口碑): 用户好评截屏合集（Twitter）
    - Day 7 (互动): 问答抽奖（社群）

### 5.2 历史数据反馈闭环
- **痛点**：AI 不知道什么样的选题在**这个**产品上表现好。
- **功能**：如果系统内有历史内容的阅读/点赞数据（或用户手动给历史计划打分），将其作为 Few-Shot Learning 的样本喂给 AI。
- **效果**：越用越懂该产品的受众口味。

### 5.3 竞品“针锋相对”模式
- **痛点**：竞品分析只停留在文档里。
- **功能**：专门针对“竞品分析”Tab 中的弱点生成选题。
- **示例**：竞品 A 价格贵，AI 自动生成“为什么你不应该为溢价买单？平替指南来了”。

### 5.4 节日/热点日历挂载
- **痛点**：用户想不起最近有什么节日。
- **功能**：系统内置公共营销日历（如程序员节、双11、春节），AI 在生成时自动检测当前日期临近的节点，主动建议结合节日的选题。

### 5.5 "标题党" 优化器
- **痛点**：选题方向对，但标题不够吸引人。
- **功能**：对已生成的选题，提供一个“优化标题”按钮，AI 提供 5 个不同风格（震惊体、疑问体、数字体、情绪体）的备选标题供 AB Test。

## 6. 技术方案概览

### 6.1 架构
- **前端**：React + Tailwind (新增 `AiTopicGenerator` 组件)。
- **后端**：Node.js / Next.js API Route (作为 Proxy)。
- **AI 服务**：火山引擎 Ark Runtime (Chat Completions API)。
- **数据库**：Supabase (`content_items` 表)。

### 6.2 数据结构扩展
无需大幅修改现有 Schema，但建议在 `content_items` 表的 `body` 字段或新增 `meta_data` JSON 字段中存储 AI 生成的“大纲”和“设计思路”，以便后续“AI 辅助撰写”功能使用。

```json
// content_items 表新增 meta_data 字段 (JSONB)
{
  "ai_generated": true,
  "topic_angle": "Efficiency/Anxiety",
  "target_persona": "Project Manager",
  "outline": ["Point 1", "Point 2", "Point 3"],
  "original_suggestion_id": "trace_id_123"
}
```

## 7. 后续规划 (Roadmap)

1.  **Phase 1 (当前)**：AI 选题助手。实现基于产品规划的选题生成与入库。
2.  **Phase 2 (下一步)**：AI 内容撰写。点击已生成的计划，AI 根据大纲自动撰写全文（文章/脚本）。
3.  **Phase 3 (进阶)**：AI 视觉生成。根据选题自动生成配图建议 Prompt（对接 Midjourney/DALL-E）。
4.  **Phase 4 (闭环)**：数据分析回填。手动或自动回填发布后的数据，优化 AI 模型偏好。

---

## 8. 验收标准
1.  **关联性**：生成的 10 个选题中，至少 8 个明显与当前产品业务相关。
2.  **多样性**：生成的选题不应雷同，覆盖至少 3 种不同的切入角度。
3.  **可用性**：从生成到入库的操作路径在 3 次点击内完成。
4.  **响应速度**：生成过程不超过 5-8 秒（流式输出体验更佳）。
