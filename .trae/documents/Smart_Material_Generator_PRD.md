# 产品需求文档：智能资料生成助手 (Smart Material Generator)

## 1. 功能名
智能资料生成助手 (Smart Material Generator)
> 参考 NotebookLM 逻辑，打造基于资料上下文的智能内容生成工具。

## 2. 需求描述
用户需要一个能够理解上传资料或对话内容，并根据特定目标（如“小红书文案”、“产品通告”、“周报”）自动生成内容的工具。核心逻辑是 **Input (资料/对话) -> Context (AI理解) -> Output (基于预设模板生成)**。

## 3. 概述
本项目旨在为 PenSoul 增加一个核心创作模块，模拟 Google NotebookLM 的体验。用户可以在左侧区域上传文档（PDF、Markdown、TXT）或与 AI 进行自由对话以补充信息；在右侧区域，系统提供一系列“生成目标卡片”（如小红书、知乎、公众号、PRD片段等）。用户选择某个目标后，AI 将基于左侧的资料上下文，结合预设的专家级 Prompt，生成符合该平台风格或格式的内容。

## 4. 相关页面设计

### 4.1 页面布局 (两栏式设计)
*   **左侧：资料与交互区 (Source & Chat)**
    *   **顶部/悬浮**: 资料管理入口。显示“上传资料”按钮，以及已上传资料的列表（Chip 或 列表形式）。支持移除资料。
    *   **主体**: 聊天对话流。用户可以像与 ChatGPT 对话一样输入信息，补充资料中未包含的细节。
    *   **底部**: 输入框。
*   **右侧：生成控制台 (Generation Dashboard)**
    *   **标题**: "基于资料生成 (Generate from sources)"
    *   **卡片网格**: 展示预设的生成目标。
        *   卡片示例：
            *   [图标] **小红书笔记** (提取亮点，生成种草文案)
            *   [图标] **FAQ 生成** (基于资料生成常见问题解答)
            *   [图标] **产品简介** (生成一句话和一段式介绍)
            *   [图标] **简报/摘要** (快速总结核心内容)
            *   [图标] **可视化图表** (生成路线图、架构图、流程图)
            *   [图标] **UI 原型** (生成简单的 HTML/Tailwind 原型界面)
    *   **结果展示区 (Notes List)**: 
        *   **列表模式**: 生成的内容不再是临时的，而是被保存为“笔记 (Note)”。这些笔记以卡片列表的形式展示在功能选择区下方。
        *   **笔记卡片**: 每张卡片包含标题（自动生成或默认标题）、类型图标（如小红书）、生成时间摘要。
        *   **查看与编辑**: 点击任意笔记卡片，进入详情视图（模态框或侧边展开）。详情页支持 Markdown 渲染查看，并提供富文本编辑器进行二次修改。
        *   **操作**: 支持“复制”、“删除”、“Pin (置顶)”、“保存到灵感集”。

### 4.2 交互细节
*   **资料解析**: 上传文件后，系统需显示解析进度。解析完成后，资料状态变为“已就绪”。
*   **上下文感知**: 用户在对话区输入的每一句话，以及上传的每一份文档，都共同构成了“当前上下文 (Current Context)”。
*   **一键生成**: 用户无需编写复杂的 Prompt，只需点击右侧卡片。系统会自动将“当前上下文” + “卡片对应的 System Prompt” 组合发送给 LLM。

## 5. 用户旅程 (User Journey)

1.  **进入模块**: 用户点击导航栏的“智能资料助手”进入页面。
2.  **导入素材**: 用户点击“上传”，选择了一份《PenSoul v2.0 功能说明书.pdf》。
3.  **补充信息**: 用户觉得文档里没写清楚目标用户，于是他在对话框输入：“补充一点，这个版本主要针对的是自媒体创作者和独立开发者。”
4.  **选择目标**: 用户看向右侧面板，发现了“小红书宣发”卡片，点击它。
5.  **获取结果**: 系统提示“正在阅读资料并生成...”，几秒后，右侧下方的笔记列表中新增了一张名为“关于 PenSoul v2.0 的小红书文案”的笔记卡片。
6.  **查看与编辑**: 用户点击卡片，弹出了详细内容。用户觉得不错，简单修改了几个 Emoji，然后点击“保存到灵感库”。

## 6. 用户故事 (User Stories)

*   **作为运营人员**，我希望上传枯燥的产品手册后，能直接生成吸引人的小红书和朋友圈文案，而不需要自己从头提炼卖点。
*   **作为产品经理**，我希望把杂乱的会议记录丢进去，然后一键生成结构清晰的“会议纪要”或“待办事项清单”。
*   **作为学习者**，我希望上传一篇长论文，然后让 AI 帮我生成“核心观点摘要”或“学习卡片”。

## 7. 实现逻辑 (Implementation Logic)

### 7.1 前端 (React)
*   **组件复用**:
    *   复用 `IdeaChat` 或 `ChatInterface` 作为左侧对话组件基础。
    *   复用 `DocumentManager` 用于文件上传 UI。
*   **新组件**:
    *   `SmartSourcePage`: 主容器。
    *   `GenerationGrid`: 右侧生成目标选择器。
    *   `NoteList`: 右侧下方展示生成历史的笔记列表组件。
    *   `NoteDetailModal`: 笔记详情与编辑组件。
    *   **可视化渲染组件**:
         *   `ArchitectureRenderer`: 基于 `React Flow` 或直接渲染 AI 生成的 `React/Tailwind` 代码，实现高保真、美观的架构图与流程图（替代 Mermaid 的默认样式）。
         *   `CodePreview`: 用于渲染 HTML/Tailwind 原型代码。
*   **状态管理**:
    *   `sources`: 数组，存储已上传文件的内容或引用。
    *   `chatHistory`: 数组，存储对话历史。
    *   `notes`: 数组，存储生成的笔记对象 (包含 id, title, content, type, timestamp)。
    *   `generationStatus`: 'idle' | 'generating' | 'success' | 'error'。

### 7.2 后端/AI 交互
*   **Prompt 组装策略**:
    *   **常规文案生成**: 按照角色设定、参考资料、用户输入生成文本。
    *   **架构图/原型生成**:
        *   Prompt 指令: "Generate a modern, beautiful architecture diagram using React and Tailwind CSS. Use flexbox/grid for layout, rounded corners, shadows, and gradients for nodes."
        *   不再输出 Mermaid 代码，而是输出完整的 React 组件代码或 HTML 片段。
*   **资料处理**: 
    *   短期方案：直接将文件文本读取后放入 Prompt (适用于长文本模型)。
    *   长期方案：RAG (检索增强生成)，将资料存入向量库，根据生成目标检索相关片段。**本期建议采用短期方案，利用长窗口模型 (如 Claude-3-200k, GPT-4-128k, DeepSeek-V3) 直接处理。**

## 8. 功能细节描述

1.  **支持文件格式**:
    *   .txt, .md (Markdown): 直接读取文本。
    *   .pdf: 需要前端或后端解析为文本 (使用 `pdf-parse` 或类似库)。
    *   .docx: (可选，二期支持)。
2.  **生成卡片配置**:
    *   系统需预埋一套 Prompt 模板库 (Templates)。
    *   每个卡片对应一个 Template ID。
    *   支持“自定义卡片”：用户可以自己写 Prompt 并保存为卡片 (高阶功能，可后置)。
3.  **上下文窗口限制**:
    *   前端需计算当前资料总字数，如果超过模型限制，给予提示或自动截断/摘要。
4.5.  **流式输出**:
    *   生成结果应支持 Stream 打字机效果，减少用户等待焦虑。
6.  **代码/图表渲染**:
    *   **架构图渲染**: 优先使用 AI 生成的 `React/Tailwind` 代码进行渲染，而非简单的 Mermaid 字符串，以实现自定义节点样式和高级布局。
    *   **原型预览**: 当生成内容为 UI 原型代码时，前端提供交互式预览。
    *   **Fallback**: 对于简单的逻辑图表，仍可保留 Mermaid 作为备选方案。
