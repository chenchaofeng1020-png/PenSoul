# 灵感孵化器 (Idea Incubator) 产品需求文档

## 1. 概述

### 1.1 背景
用户在日常生活中经常会迸发一些突发奇想，这些想法可能是新产品的雏形，也可能是对现有问题的创新解决方案。然而，这些灵感往往转瞬即逝，或者因为缺乏系统的梳理而停留在“空想”阶段。

### 1.2 目标
打造一个轻量级的“灵感孵化器”，帮助用户：
1.  **快速捕捉**：随时随地记录灵感。
2.  **深度梳理**：通过与 AI 对话，将模糊的想法结构化、具体化。
3.  **无缝转化**：将成熟的想法一键转化为正式的产品规划项目。

### 1.3 用户画像
-   **产品经理**：需要记录和验证新功能或新产品想法。
-   **创业者**：需要快速梳理商业模式和产品定位。
-   **独立开发者**：需要评估技术可行性和市场需求。

## 2. 功能模块

### 2.1 入口
-   **位置**：左上角产品切换下拉菜单中。
-   **样式**：新增“灵感工作台”或“Idea Incubator”选项，配以独特的图标（如灯泡 💡）。

### 2.2 灵感列表 (Idea List)
-   **功能**：展示用户记录的所有灵感。
-   **展示项**：
    -   标题（或想法的第一句话）。
    -   创建时间。
    -   状态（孵化中、已转化、已归档）。
    -   最后更新时间。
-   **操作**：
    -   新建灵感。
    -   点击进入详情/对话页。
    -   删除/归档。

### 2.3 灵感对话页 (Idea Chat)
核心功能区，用户与 AI 在此进行交互。

#### 2.3.1 初始引导
-   当用户新建灵感时，AI 主动问候：“你有什么新想法？请告诉我，我来帮你一起梳理。”
-   用户输入第一段描述后，AI 根据内容进行初步分析，并提出引导性问题。

#### 2.3.2 AI 角色设定 (Persona)
-   **角色**：资深产品顾问 & 创业导师。
-   **能力**：
    -   **提问**：引导用户思考目标用户、痛点、解决方案、竞品差异、商业模式等。
    -   **总结**：在对话过程中，实时提炼关键信息（如：“听起来你的目标用户是...，核心痛点是...”）。
    -   **建议**：基于行业知识提供补充建议（如：“在这个领域，你可能需要关注...竞品”）。

#### 2.3.3 对话交互
-   支持纯文本输入。
-   支持 Markdown 渲染（AI 回复可能包含列表、表格）。
-   **结构化预览**（侧边栏或浮层）：
    -   在对话过程中，AI 会自动提取结构化信息并在侧边展示（实时更新）。
    -   字段包括：产品名称、Slogan、目标用户、核心痛点、解决方案、关键功能、商业模式。

### 2.4 转化为产品 (Convert to Product)
-   **触发条件**：当结构化信息积累到一定程度，或用户主动点击“转为产品”。
-   **流程**：
    1.  用户点击“转为产品”按钮。
    2.  系统弹窗确认转化后的产品名称。
    3.  系统将“灵感对话页”中提炼的结构化数据，自动映射到“产品规划”模块的对应字段。
    4.  自动创建一个新的产品，并跳转到该产品的规划页面。
    5.  原灵感状态更新为“已转化”，并关联到新产品（保留历史记录）。

## 3. 详细需求

### 3.1 界面布局
-   **左侧列表**：灵感历史记录列表。
-   **中间对话区**：类似 ChatGPT/Claude 的对话界面。
-   **右侧结构化面板**（可折叠）：
    -   **Idea Memo**：AI 自动整理的笔记。
    -   **转化按钮**：常驻底部或顶部。

### 3.2 数据结构 (Draft)
```json
{
  "id": "idea_123",
  "title": "关于宠物智能喂食器的想法",
  "created_at": "2023-10-27T10:00:00Z",
  "status": "incubating", // incubating, converted, archived
  "chat_history": [
    { "role": "ai", "content": "..." },
    { "role": "user", "content": "..." }
  ],
  "structured_data": {
    "product_name": "PetFeed Pro",
    "target_audience": "忙碌的宠物主人",
    "pain_points": ["无法按时喂食", "不知道宠物进食量"],
    "solutions": ["APP远程控制", "摄像头监控", "健康数据分析"]
  }
}
```

### 3.3 补充创意点
1.  **SWOT 分析生成**：在梳理后期，可以请求 AI 生成一份 SWOT 分析。
2.  **Pitch Deck 大纲**：转化前，可以生成一份简易的融资路演 PPT 大纲。
3.  **情绪板 (Mood Board)**：允许用户上传参考图片，AI 分析视觉风格（V2阶段）。

## 4. 交互流程图 (文字版)

1.  **入口**：用户点击左上角下拉 -> 选择“灵感工作台”。
2.  **新建**：进入工作台 -> 点击“+ 新建灵感” -> 输入想法（如“我想做一个针对老人的智能药盒”）。
3.  **对话**：
    -   AI：“这个想法很棒！你心目中的老人主要是有哪些特征？是独居老人还是有子女同住？”
    -   用户：“主要是独居老人，容易忘记吃药。”
    -   AI：“明白了。针对独居老人的健忘痛点。那你觉得除了提醒，还需要什么功能让子女放心？”
    -   （对话持续... 右侧面板实时更新：目标用户-独居老人；痛点-健忘...）
4.  **转化**：
    -   用户觉得差不多了，点击“转为产品”。
    -   系统提示：“已为你生成产品‘智能药盒’，正在跳转...”。
    -   跳转至新产品的“产品规划”页，所有字段已自动填充。
