# 产品路线图功能设计（Roadmap）

## 1. 背景与目标
- 清晰呈现产品的中长期规划，提升跨团队对齐效率。
- 以“可视化+结构化数据”为核心：不同视图满足不同场景（季度/时间轴/泳道/Now-Next-Later）。
- 支持协作、版本快照与变更追踪，确保路线图可信、可回溯。

## 2. 用户角色与典型场景
- 产品经理：规划路线、拆分里程碑与特性、维护依赖、发布路线图。
- 研发负责人：评估排期与资源，更新进度与风险。
- 设计/运营：查看版本节点，跟进上线准备。
- 管理层：审阅路线图，确认优先级与目标达成。

## 3. 信息架构与导航
- 在侧边栏新增“产品路线图”入口（列表页+详情页）。
- 路线图页面包含：视图切换、筛选与分组、里程碑/版本、依赖关系、评论、快照与导出。

## 4. 核心功能设计
1) 多视图
- 季度视图：按 Q1/Q2/Q3/Q4 展示，适合管理层审阅。
- 时间轴视图：横轴为时间，卡片显示特性与阶段。
- 泳道视图：按团队/平台/目标泳道分列，适合跨团队协作。
- Now-Next-Later：战略分层，适合早期规划与对外沟通。

2) 筛选与分组
- 按产品线、团队、优先级、状态、负责人、标签、里程碑/版本。
- 支持保存筛选器与共享视图。

3) 里程碑与版本管理
- 里程碑：目标、日期、进度、风险、负责人。
- 版本：版本号、发布窗、关联特性列表。

4) 依赖关系与冲突检测
- Item 间可设依赖（前置/并行/阻塞）。
- 冲突提示：时间重叠、资源超载、跨团队阻塞。

5) 进度与状态
- 状态枚举：规划中/设计中/开发中/测试中/待发布/已发布/搁置。
- 进度条：0–100%，支持自动/手动更新。

6) 协作与变更留痕
- 评论@人、变更说明、讨论串。
- 变更留痕与通知：支持记录变更历史、@提醒与订阅通知。

7) 版本管理与快照
- 每次重大变更形成快照，支持对比与回看（只读）。

8) 导出与分享
- 导出 SVG/PDF（用于汇报），CSV（用于分析）。
- 受权限控制的分享链接。

## 5. 数据模型（核心表与字段）
- roadmap (路线图)：id, name, product_line, owner, time_frame, description, created_at, updated_at
- roadmap_item (条目)：id, roadmap_id, title, type(feature/milestone/version), status, start_date, end_date, quarter, team, owner, priority, risk, effort, progress, milestone_id, release_version, tags(JSON), links(JSON), description, created_at, updated_at
- roadmap_dependency：id, from_item_id, to_item_id, relation_type(blocks/precedes/parallel)
- roadmap_version_snapshot：id, roadmap_id, snapshot_name, data(JSON), created_by, created_at
- roadmap_comment：id, roadmap_id, item_id, author_id, content, created_at
- roadmap_activity_log（可选）：id, target_type(roadmap/item), target_id, actor_id, action, data(JSON), created_at

## 6. API 设计（REST 示例）
- GET /api/roadmaps?filters=... → 列表与聚合统计
- POST /api/roadmaps → 新建路线图
- GET /api/roadmaps/{id} → 详情与里程碑/版本/依赖
- PATCH /api/roadmaps/{id} → 更新基础信息
- POST /api/roadmaps/{id}/items → 批量/单条新增条目
- PATCH /api/roadmap-items/{id} → 更新条目（状态/进度/日期/负责人等）
- POST /api/roadmap-items/{id}/dependencies → 新增依赖
- DELETE /api/roadmap-dependencies/{id} → 删除依赖
- POST /api/roadmaps/{id}/snapshots → 生成快照
- GET /api/roadmaps/{id}/snapshots → 快照列表与对比
- POST /api/comments → 评论新增；GET /api/comments?roadmapId=... → 评论列表
- GET /api/activity-logs?targetType=roadmap&targetId=...（可选）→ 变更历史

认证与权限：沿用 JWT，角色（PM/Lead/Viewer/Admin）控制增删改查与访问权限。

## 7. 前端交互与组件规划
- 列表页：筛选条(RoadmapFilterBar)、集合统计、路线图卡片(RoadmapCard)。
- 详情页：视图切换(Tab/Segmented)、时间轴/泳道组件(RoadmapTimeline/RoadmapSwimlane)、条目卡(RoadmapItemCard)、依赖视图(RoadmapDependencyGraph)。
- 编辑：新建/编辑弹窗(RoadmapEditModal)、拖拽排序、批量选择操作、评论(CommentsPane)、通知订阅与更新提示。
- 视觉要点：颜色编码（状态/优先级）、图例、空状态与引导、可读的日期/季度标签。

## 8. 验收标准（示例）
- 能以季度/时间轴/泳道/Now-Next-Later 四种视图展示同一数据集。
- 支持筛选与分组，并能保存共享视图。
- Item 支持依赖配置与冲突提示。
- 路线图可生成快照并对比差异。
- 变更可追溯、通知可达。
- 导出 SVG/PDF 成功，信息完整、布局合理。

## 9. 实施计划
- 里程碑 M1：数据模型与 API（2 周）
- 里程碑 M2：详情页与时间轴/泳道视图（2 周）
- 里程碑 M3：依赖/评论/协作与通知（2 周）
- 里程碑 M4：快照与导出（1 周）
- 里程碑 M5：性能优化与可用性改进（1 周）

## 10. 风险与缓解
- 数据一致性：快照与当前状态不一致 → 事务与乐观锁；变更审计。
- 资源排期冲突：提供冲突检测与提示规则。
- 可用性：复杂视图学习成本高 → 空状态、渐进引导与示例数据。

## 11. 成功指标
- 路线图覆盖率（含关键里程碑与版本）。
- 路线图更新频次与协作活跃度。
- 冲突发现并解决的平均时长。
- 管理层审阅效率与满意度反馈。
