# 人天评估功能需求文档

## 1. 功能概述

### 1.1 功能名
AI 智能人天评估系统

### 1.2 需求描述
为用户提供基于 AI 的项目需求工作量评估功能。用户可导入客户需求清单或需求说明作为数据源，AI 根据功能列表逐条评估产品、UI、前端、后端、测试五个角色的人天工作量，最终生成交互式网页报告，支持在线查看和修改。

### 1.3 概述
在项目管理中，准确评估开发工作量是项目规划的关键环节。本功能利用 AI 能力，结合行业标准和中级人员水平，为每个功能点提供多角色的人天评估，帮助项目经理快速制定项目计划和资源分配方案。

### 5.2 核心模块设计

#### 模块 1：数据源解析器 (DataSourceParser)
- **输入**: 用户选择的 sources (文档、表格、文本)
- **处理**: 
  - 根据文件类型选择解析策略
  - Markdown/TXT: 正则提取功能点
  - Excel/CSV: 读取表格数据
  - 图片: OCR 识别 (可选)
- **输出**: 标准化的功能列表 `[{id, name, description, module}]`

#### 模块 2：分批处理器 (BatchProcessor)
- **输入**: 功能列表、批次大小
- **处理**:
  - 按批次大小切分功能列表
  - 维护处理进度状态
  - 支持断点续传
- **输出**: 批次队列，每批包含若干功能

#### 模块 3：AI 评估引擎 (AIEstimationEngine)
- **输入**: 单批功能列表 + 评估 Skill + 用户提示词
- **处理**:
  - 构建完整 Prompt
  - 调用 AI 接口
  - 解析结构化输出
- **输出**: 评估结果 `[{functionId, estimates: {product, ui, frontend, backend, test}}]`

#### 模块 4：报告生成器 (ReportGenerator)
- **输入**: 完整评估结果
- **处理**:
  - 统计各维度数据
  - 生成 Markdown 报告
  - 转换为交互式 HTML
- **输出**: 可交互的网页报告

### 6.1 人天评估 Skill 设计

```markdown
# Skill: 软件功能人天评估专家

## 角色定义
你是一位经验丰富的项目管理专家，擅长软件项目工作量评估。
你熟悉各角色（产品、UI、前端、后端、测试）的工作内容和效率标准。

## 评估标准
基于【中级人员水平】，按照 1 个人干活的标准进行评估。

### 各角色工作范围

1. **产品 (Product)**
   - 需求沟通与确认
   - 产品方案策划
   - 原型设计
   - 需求文档编写 (PRD)
   - 方案评审与修改
   - 开发过程答疑
   - 验收测试

2. **UI 设计 (UI)**
   - 视觉风格定义
   - 页面视觉设计
   - 设计规范制定
   - 设计评审与修改
   - 设计交付与标注
   - 开发支持

3. **前端开发 (Frontend)**
   - 技术方案设计
   - 页面结构与样式实现
   - 交互逻辑开发
   - 接口对接
   - 兼容性处理
   - 自测与修复

4. **后端开发 (Backend)**
   - 技术方案设计
   - 数据库设计
   - 接口开发
   - 业务逻辑实现
   - 性能优化
   - 单元测试

5. **测试 (QA)**
   - 测试用例设计
   - 功能测试执行
   - Bug 提交与验证
   - 回归测试
   - 测试报告

## 评估原则

1. **实事求是**: 根据功能复杂度合理评估，不夸大不压缩
2. **包含完整流程**: 评估包含设计、开发、测试、修复全流程
3. **考虑沟通成本**: 包含必要的会议、评审、沟通时间
4. **预留缓冲**: 复杂功能适当预留 10-20% 缓冲时间

### 6.2 数据结构定义

```typescript
// 功能项
interface FunctionItem {
  id: string;
  name: string;
  description?: string;
  module?: string;
}

// 人天评估结果
interface EstimationResult {
  functionId: string;
  functionName: string;
  module: string;
  estimates: {
    product: number;
    ui: number;
    frontend: number;
    backend: number;
    test: number;
  };
  explanation?: string;
}

// 评估配置
interface EstimationConfig {
  roles: ('product' | 'ui' | 'frontend' | 'backend' | 'test')[];
  level: 'junior' | 'middle' | 'senior';
  batchSize: number;
  customPrompt?: string;
}

// 评估报告
interface EstimationReport {
  id: string;
  productId: string;
  createdAt: string;
  config: EstimationConfig;
  results: EstimationResult[];
  summary: {
    totalFunctions: number;
    totalDays: number;
    roleTotals: Record<string, number>;
  };
}
```

### 6.3 分批处理策略

```
场景：用户导入 50 个功能的 Excel 表格

处理流程：
1. 解析 Excel 得到 50 个功能列表
2. 用户设置分批大小为 10
3. 系统将功能分成 5 批：
   - 批次 1: 功能 1-10
   - 批次 2: 功能 11-20
   - 批次 3: 功能 21-30
   - 批次 4: 功能 31-40
   - 批次 5: 功能 41-50
4. 逐批调用 AI 评估：
   - 每批构建独立 Prompt
   - 包含该批所有功能信息
   - 返回该批评估结果
5. 实时更新进度：
   - 每完成一批更新 UI
   - 显示已完成/总数
   - 显示当前处理的功能
6. 合并所有批次结果
7. 生成最终报告
```
