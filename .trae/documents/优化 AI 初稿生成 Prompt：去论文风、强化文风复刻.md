经过分析，导致“文章像论文、文风复刻不生效”的原因主要在 `server/index.cjs` 中的 Prompt 构建逻辑：

1.  **System Prompt 角色设定过于传统**：例如微信公众号设为“运营专家”，要求“逻辑严密”、“适合深度阅读”，这容易诱导 AI 写出学术风。
2.  **文风复刻权重不足**：`userStyle` 虽然被添加到了 System Prompt 中，但只是作为补充说明，很容易被前面的平台默认 Prompt 覆盖。
3.  **缺乏“口语化”强指令**：没有明确禁止“教科书式表达”或“翻译腔”。

**优化方案：**

我将修改 `server/index.cjs` 中的 `/api/ai/generate-draft` 接口逻辑：

1.  **升级 System Prompt**：
    *   **角色重塑**：将“运营专家”改为“10w+ 爆款文案主理人”，强调“人感”和“情绪价值”。
    *   **通用反内卷指令**：所有平台都增加“拒绝爹味”、“拒绝翻译腔”、“多用短句”、“像朋友聊天”等通用约束。
2.  **强化文风复刻**：
    *   将 `userStyle` 的处理逻辑移到 Prompt 的更显眼位置（如开头或作为独立的重要指令块）。
    *   增加 Few-Shot 引导词（如“请先分析该风格的断句、用词习惯，并在生成时进行 1:1 像素级复刻”）。
3.  **平台 Prompt 定制化增强**：
    *   **小红书**：强制要求第一段必须有“Hook”（钩子），文中必须有情绪词。
    *   **公众号**：从“深度阅读”改为“观点犀利 + 故事化叙述”。

**修改范围**：仅涉及 `server/index.cjs` 文件。
