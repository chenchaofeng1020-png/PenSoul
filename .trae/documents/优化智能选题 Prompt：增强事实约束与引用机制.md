经过分析，目前的 Prompt 存在以下问题，导致生成结果“脱离事实”或“胡说八道”：

**问题分析：**
1.  **缺少“忠实于素材”的约束**：当前的 System Prompt 仅要求“基于提供的素材”，但没有明确禁止 AI “捏造事实”或“过度发散”。
2.  **上下文缺失**：如果素材本身很简短（例如只有一句话），AI 为了凑字数或生成 3 个方案，很容易产生幻觉（Hallucination）。
3.  **温度值 (Temperature) 偏高**：代码中设置为 `0.8`，这增加了随机性和创造性，但对于需要基于事实的选题生成来说，可能过高了。

**优化方案：**
我将修改 `server/index.cjs` 中的 Prompt 逻辑，进行以下优化：

1.  **增强约束指令**：在 System Prompt 中加入强约束，明确要求“**核心事实必须源自素材**，严禁捏造数据或不存在的细节”。
2.  **引入“素材扩充”机制（Prompt 层）**：如果素材包含 URL（通常是文章链接），提示 AI 尝试理解该链接可能代表的上下文（虽然 AI 无法直接访问外网，但可以通过标题/摘要推断）。
3.  **调整参数**：将 `temperature` 从 `0.8` 降低到 **0.7**，在创意与准确性之间取得更好平衡。
4.  **优化输出结构**：在 JSON 输出中增加 `source_quote`（引用原文）字段，强制 AI 找到素材中的依据，从而减少胡编乱造。

这是一个 Prompt Engineering 层面的优化，无需修改前端代码。请确认执行。
