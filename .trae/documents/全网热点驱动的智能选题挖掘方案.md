# 产品需求文档 (PRD) v2.0：全网热点驱动的智能选题系统

## 1. 项目背景与目标
### 1.1 核心价值
解决内容创作者（产品经理/运营/自媒体）“不知道写什么火”的痛点。通过聚合全网热点，利用 AI 进行“热点-素材-产品”的深度连接分析，最终产出高质量的选题计划。

### 1.2 最终交付物
用户通过本功能，最终将在 **“内容规划 (Content Plan)”** 模块中生成一条状态为 `Draft`（草稿）或 `Planned`（已排期）的 **选题计划**。

## 2. 核心功能架构 (三层漏斗)

### Layer 1: 趋势发现 (Trend Discovery)
*   **入口**：侧边栏新增“趋势雷达”入口。
*   **多源榜单**：
    *   **知乎** (深度观点)
    *   **微博** (社会热点)
    *   **36Kr** (科技商业)
    *   **B站/抖音** (年轻趋势)
*   **展示**：排名、标题、热度值、分类标签。支持点击进入 Layer 2。

### Layer 2: 素材深度解构 (Material Deep Dive)
*   **素材流**：针对选中热点，展示 3-5 篇核心素材（摘要/评论）。
*   **PAS-C 智能分析**：
    *   **P (Phenomenon)**: 事实概括。
    *   **A (Angle)**: 切入视角。
    *   **S (Sentiment)**: 情绪洞察。
    *   **C (Connect)**: **关键连接点**（热点与用户产品的结合逻辑）。
*   **交互**：用户勾选认可的分析点，点击“生成选题”。

### Layer 3: 选题落地 (Idea Execution) —— **核心闭环**
*   **生成结果**：AI 输出 1-3 个选题方案（标题、核心观点、大纲）。
*   **创建计划 (Create Plan)**：
    *   用户选中满意的方案，点击 **“创建选题计划”** 按钮。
    *   **系统动作**：
        1.  调用 `ContentService.createItem` 接口。
        2.  **字段映射**：
            *   `Title` -> 选题标题。
            *   `Description` -> 选题核心观点 + 来源热点。
            *   `Content` (RichText) -> AI 生成的大纲 + 原始素材链接。
            *   `Status` -> `Draft` (草稿)。
            *   `Platform` -> 根据热点来源自动推荐（如知乎热点推荐发知乎）。
        3.  **跳转**：自动跳转到“内容规划”页面，并高亮显示新创建的这条计划，方便用户进一步编辑排期。

## 3. 详细交互设计

### 3.1 趋势雷达页
*   **双栏布局**：左侧热榜，右侧分析详情。
*   **右侧详情页**：
    *   顶部：热点标题 + 原始链接。
    *   中间：素材卡片流（左文右析）。
    *   底部固定栏：**[生成选题方案]** 按钮。

### 3.2 方案确认弹窗
点击底部按钮后，弹出“选题方案预览”：
*   展示 3 个不同角度的卡片（例如：借势型、深度型、反讽型）。
*   用户点击某个卡片上的 **“采用并创建”**。

## 4. 数据与逻辑

### 4.1 PAS-C 分析模型
*   **Input**: 素材文本 + `CurrentProduct` (名称/定位/受众)。
*   **Output**: JSON `{ phenomenon, angle, sentiment, connection_point }`。

### 4.2 落地数据结构
最终生成的 Content Item 数据结构示例：
```json
{
  "title": "从美团改版看SaaS的温度：别让算法异化了人",
  "type": "article",
  "status": "draft",
  "platform": "zhihu",
  "content": "<h2>核心观点</h2><p>效率不是唯一指标...</p><h2>大纲</h2><ul><li>1. 现象...</li><li>2. 分析...</li></ul><hr><p>来源热点：美团众包改版</p>",
  "meta": {
    "source_trend_id": "trend_123",
    "ai_generated": true
  }
}
```

## 5. 验收标准
1.  **完整链路**：用户从点击热点开始，必须能走通“看素材 -> 看分析 -> 选方案 -> 创建计划”的全流程。
2.  **数据一致性**：创建的计划必须能在“内容规划”列表中找到，且内容完整。
3.  **上下文关联**：生成的选题必须体现用户当前产品的特性，而非通用文章。
