# 内容规划功能落地版方案

## 1. 核心问题定义

### 现状问题
- 当前只有简单的内容日历，功能太单薄
- 用户只能手动添加内容，缺乏数据支撑
- 无法分析竞品内容策略
- 协作功能缺失，团队无法高效配合

### 具体需求（可落地）
1. **竞品内容监控** - 手动添加竞品，追踪其内容动态
2. **内容库管理** - 建立可复用的内容素材库
3. **团队协作** - 多人编辑、审核、发布流程
4. **数据看板** - 基础的数据统计和对比

## 2. 具体功能设计（第一版）

### 2.1 竞品内容监控模块

**竞品管理**
```
功能流程：
1. 手动添加竞品（输入竞品名称、选择平台）
2. 系统定期抓取竞品公开内容（每日一次）
3. 展示竞品内容列表和基础数据
4. 支持手动刷新获取最新内容

具体实现：
- 竞品添加表单：名称、平台选择、描述
- 内容抓取：调用各平台公开API或爬虫
- 数据存储：竞品信息表 + 竞品内容表
- 展示方式：列表+卡片视图
```

**竞品内容分析**
```
基础统计：
- 近7天/30天发布数量
- 内容类型分布（图文/视频/文章）
- 平均互动数据（点赞/评论/转发）
- 高频关键词提取（简单词频统计）

展示形式：
- 简单的柱状图和饼图
- 表格展示详细数据
- 支持导出Excel
```

### 2.2 内容库管理模块

**内容素材库**
```
功能设计：
1. 创建内容素材（标题、正文、标签、类型）
2. 支持图片/视频上传
3. 标签分类管理
4. 搜索和筛选功能

实用细节：
- 支持富文本编辑器（基础版）
- 标签系统：手动添加标签
- 使用记录：记录素材使用历史
- 复用统计：统计使用次数
```

**内容模板**
```
模板类型：
- 产品推广模板
- 活动预告模板
- 品牌故事模板
- 用户案例模板

模板功能：
- 预设标题格式
- 正文结构框架
- 配图建议位置
- 适用平台说明
```

### 2.3 团队协作功能

**基础权限管理**
```
角色定义：
- 管理员：全部权限
- 编辑：创建、编辑内容
- 审核：审核内容、提出修改意见
- 发布：发布已审核内容

权限控制：
- 基于角色的功能访问控制
- 内容状态权限（草稿/审核中/已发布）
- 操作日志记录
```

**内容工作流**
```
状态流转：
草稿 → 待审核 → 审核通过 → 已发布
   ↓        ↓         ↓
已拒绝   需修改   已下线

协作功能：
- 评论和批注系统
- @提醒功能
- 修改历史对比
- 审核意见记录
```

### 2.4 数据看板模块

**基础统计**
```
内容统计：
- 总发布数量（按月/周）
- 各平台发布分布
- 内容类型占比
- 发布准时率

团队统计：
- 成员发布数量排行
- 审核通过率
- 平均审核时长
- 任务完成率
```

**竞品对比**
```
对比维度：
- 发布频率对比
- 内容类型对比
- 平均互动对比
- 关键词重合度

展示方式：
- 并排对比表格
- 简单的趋势图
- 差异高亮显示
```

## 3. 技术实现方案（务实版）

### 3.1 数据库设计

```sql
-- 竞品表
CREATE TABLE competitors (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  platforms TEXT[], -- 平台数组
  description TEXT,
  website_url VARCHAR(255),
  created_at TIMESTAMP DEFAULT NOW()
);

-- 竞品内容表
CREATE TABLE competitor_contents (
  id SERIAL PRIMARY KEY,
  competitor_id INTEGER REFERENCES competitors(id),
  platform VARCHAR(50),
  content_type VARCHAR(20), -- video/article/image
  title VARCHAR(255),
  content_text TEXT,
  content_url VARCHAR(500),
  publish_time TIMESTAMP,
  like_count INTEGER DEFAULT 0,
  comment_count INTEGER DEFAULT 0,
  share_count INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 内容素材表
CREATE TABLE content_assets (
  id SERIAL PRIMARY KEY,
  title VARCHAR(255) NOT NULL,
  content TEXT,
  content_type VARCHAR(20),
  tags TEXT[],
  images TEXT[], -- 图片URL数组
  usage_count INTEGER DEFAULT 0,
  created_by INTEGER,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 内容计划表（扩展）
CREATE TABLE content_plans (
  id SERIAL PRIMARY KEY,
  title VARCHAR(255) NOT NULL,
  content TEXT,
  platform VARCHAR(50),
  schedule_time TIMESTAMP,
  status VARCHAR(20) DEFAULT 'draft',
  created_by INTEGER,
  assigned_to INTEGER,
  asset_id INTEGER REFERENCES content_assets(id),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- 审核记录表
CREATE TABLE review_records (
  id SERIAL PRIMARY KEY,
  content_id INTEGER REFERENCES content_plans(id),
  reviewer_id INTEGER,
  action VARCHAR(20), -- approve/reject/request_change
  comment TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### 3.2 接口设计（简化版）

```javascript
// 竞品相关接口
GET /api/competitors - 获取竞品列表
POST /api/competitors - 添加竞品
GET /api/competitors/:id/contents - 获取竞品内容
POST /api/competitors/:id/refresh - 手动刷新竞品内容

// 内容素材接口
GET /api/content-assets - 获取素材列表
POST /api/content-assets - 创建素材
PUT /api/content-assets/:id - 更新素材
GET /api/content-assets/search - 搜索素材

// 协作相关接口
POST /api/content-plans/:id/submit - 提交审核
POST /api/content-plans/:id/review - 审核内容
GET /api/content-plans/:id/history - 获取修改历史
POST /api/content-plans/:id/comment - 添加评论

// 统计接口
GET /api/stats/content - 内容统计
GET /api/stats/team - 团队统计
GET /api/stats/competitor-comparison - 竞品对比
```

## 6. 风险控制

### 技术风险
- **抓取限制**: 控制抓取频率，避免被封IP
- **数据准确性**: 手动录入+自动抓取结合
- **性能问题**: 分页加载+缓存优化

### 业务风险
- **用户接受度**: 先做MVP，收集反馈迭代
- **竞品变化**: 定期维护竞品列表
- **平台政策**: 关注各平台API政策变化
