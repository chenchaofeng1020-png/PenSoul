## 平台配置字段（示例）
- id：平台唯一标识（如 `xiaohongshu`）
- name：展示名称（如“小红书”）
- category：`social` | `short_video` | `content_account` | `tech_community` | `forum`
- contentKinds：支持内容类型集合（`article` | `short_post` | `image_post` | `video`）
- constraints：
  - titleMax / bodyMax
  - imageMaxCount / videoRequired / linkAllowed / hashtagAllowed
- icon：前端使用的图标键（与现有图标库/自定义 SVG 对应）
- urlPatterns：用于归档“实际发布链接”的识别/校验（正则）
- color：平台主题色（UI 标识用）

示例：
```json
{
  "id": "xiaohongshu",
  "name": "小红书",
  "category": "social",
  "contentKinds": ["image_post", "short_post"],
  "constraints": {"titleMax": 30, "bodyMax": 1000, "imageMaxCount": 9, "linkAllowed": true, "hashtagAllowed": true},
  "icon": "redbook",
  "color": "#ff2442",
  "urlPatterns": ["^https?://www.xiaohongshu.com/.+"]
}
```

## 数据落地与前端改造
1. 新增常量文件 `src/constants/platforms.ts`：导出平台配置数组与 `Record<id, config>`。
2. 内容规划页面（`ContentPlanningPage`）的“平台选择器”改为读取常量文件；平台筛选同样使用常量。
3. `content_items.platform` 字段保存 `id`（枚举值），展示时映射 name/icon/color。
4. 新建计划弹窗校验：根据平台 `constraints` 校验标题/正文长度与素材数量、类型。
5. 详情抽屉（后续）展示平台标识与主题色，支持打开 `actual_url` 进行验证。

## Supabase 侧
- 无需新增表；`content_items.platform` 保持 `text`，前端严格使用枚举；如需约束可在后续增加 `CHECK (platform in ...)`。
- 可增加视图 `content_items_view` 用于聚合平台名（后续再评估）。

## 扩展与维护
- 平台配置按文件维护，支持快速增删；后续若接入平台 API（如微信公众号/知乎专栏），在配置增加 `apiCapable: true` 并附 `authType`（OAuth/凭证）。
- 约束值来源于平台公开规则，后续根据平台调整（例如微博正文长度上限、抖音仅视频等）。
