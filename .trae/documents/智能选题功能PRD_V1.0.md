# 产品需求文档 (PRD)：智能选题工作台 (Smart Topic Workbench)

**版本**: V1.0  
**状态**: 待确认  
**核心理念**: 将成熟写手的“选题方法论”系统化，从“灵感乍现”转变为“工业化筛选与加工”。

---

## 1. 核心设计思路 (Methodology to Product)

基于《智能选题需求.md》，我们将系统设计为 **“采集 -> 筛选 -> 加工 -> 验证”** 的流水线。

| 方法论概念 | 对应系统模块 | 功能描述 |
| :--- | :--- | :--- |
| **情报网络** | **灵感收集箱 (Inspiration Inbox)** | 聚合热点、用户痛点（评论/私信）、竞品爆款链接。 |
| **深度加工** | **角度实验室 (Angle Lab)** | 针对同一素材，利用 AI 强制生成 5 种不同维度的切入点（反直觉、细分等）。 |
| **三叶草模型** | **选题评分器 (Topic Scorer)** | AI 辅助评估选题的“痛点程度”、“热度”和“人设匹配度”。 |
| **成文框架** | **大纲生成器 (Framework Builder)** | 选定选题后，生成包含金句、案例、逻辑流（现象、观点、分析逻辑、论据、结论等）的深度大纲。 |

---

## 2. 功能模块详细设计

### 模块一：灵感收集箱 (Inspiration Inbox)
**目标**：解决“写什么”的源头问题，建立素材池。

*   **1.1 热点雷达 (现有功能升级)**
    *   *现状*：已有全网热榜。
    *   *升级*：增加**“一键入库”**功能。用户在热点雷达看到感兴趣的话题，点击“加入选题池”，不再强制立即生成，而是先囤积。
*   **1.2 痛点录入 (新增)**
    *   *功能*：一个快速记录板。
    *   *输入*：支持粘贴“用户评论”、“私信提问”或“搜索关键词”。
    *   *AI 处理*：自动提炼背后的“核心焦虑”或“需求标签”。
    *   *场景*：用户看到一条扎心的评论，随手复制进来。
*   **1.3 爆款拆解 (新增)**
    *   *功能*：粘贴竞品/爆款文章链接。
    *   *AI 处理*：自动提取其表达的现象、核心观点、分析逻辑、论据、结论、写作手法和用户情绪点，思考它为什么好？是标题好？是切入点新？还是情绪共鸣强？不足之处是什么我可以避免。

### 模块二：角度实验室 (Angle Lab) - **核心差异化功能**
**目标**：解决“切入点平庸”的问题。

*   **交互流程**：
    1.  用户从“灵感收集箱”中拖拽/选择 1-3 个素材。
    2.  用户选择（或 AI 推荐）加工模式：
        *   🧬 **反常识模式**：生成挑战认知的观点（如“为什么我不建议...”）。
        *   🔬 **极致细分模式**：限定特定人群/场景（如“30岁职场宝妈...”）。
        *   📋 **清单/盘点模式**：提供高密度信息增量。
        *   🔥 **情绪共鸣模式**：侧重情感宣泄。
        *   🤝 **借势关联模式**：强行关联当下最火的热点。
    3.  **输出**：系统生成 3-5 个**选题卡片**。
        *   *卡片内容*：【标题】+【核心立意】+【推荐理由（基于三叶草模型）】。

### 模块三：决策与大纲 (Decision & Structuring)
**目标**：解决“如何写”和“是否值得写”的问题。

*   **3.1 选题验证 (The Check)**
    *   在最终确定前，弹窗进行**“灵魂三问”**（用户自检 + AI 辅助）：
        1.  覆盖面评分（AI 基于关键词热度估算）。
        2.  痛点深度（AI 分析情绪激烈程度）。
        3.  新意自评（用户勾选：是否有增量信息？）。
*   **3.2 深度框架生成**
    *   选中一个选题后，生成**结构化大纲**：
        *   **标题**：与选题卡片中一致？。
        *   **开篇 (Hook)**：如何用 3 秒钟抓住用户（提问/金句/反差）。
        *   **正文 (Body)**：描述现象、核心观点、分析逻辑、论据、结论。
        *   **结尾 (Call to Action)**：升华主题，引导互动。

## 3. 用户使用路径 (User Journey)

1.  **闲逛/收集**：早上刷热点雷达，看到“AI 取代程序员”热搜 -> **加入灵感箱**。下午看到粉丝留言“35岁焦虑” -> **录入灵感箱**。
2.  **策划**：打开“角度实验室”，勾选上述两个素材。
3.  **加工**：选择“情绪共鸣”和“反常识”模式。
4.  **决策**：系统生成标题《35岁程序员的退路，不是转行，而是...》。用户觉得不错。
5.  **输出**：点击“生成大纲”，获得全文结构，开始写作。

## 4. 开发优先级 (MVP)

1.  **P0 (最高)**：**角度实验室**。这是目前最缺少的“中间层”逻辑。需要改造 `generate-idea` 接口，支持多种 Prompt 模式。
2.  **P1**：**深度框架生成**。新增 `generate-framework` 接口。
3.  **P2**：灵感收集箱（UI 层面整合热点和手动录入）。
