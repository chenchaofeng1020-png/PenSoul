## 目标
- 将登录、注册流程中所有可能出现的错误提示统一转为中文，不再直接显示英文 `error.message`。
- 保持现有 UI（`Toast`、表单错误区）和逻辑不变，仅集中做错误消息的中文化。

## 改动范围
- 新增错误翻译工具：`src/services/authErrors.js`，提供 `translateAuthError(message)`。
- 接入点：
  - `src/services/api.js:61` 注册失败时抛错 -> 使用 `translateAuthError(error.message)`。
  - `src/services/api.js:109` 登录失败时抛错 -> 使用 `translateAuthError(error.message)`。
  - `src/App.jsx:105` 登录/注册 `catch` 中 `setError(error.message)` -> 以 `translateAuthError(error.message)` 设置。
- 开发测试页（可选）：`test_login_flow.html` 的英文文案替换为中文。

## 错误映射（示例）
- `Invalid login credentials` → `账号或密码错误`
- `Email not confirmed`/`Email confirmation required` → `邮箱尚未验证，请先完成邮箱验证`
- `User already registered`/`Email already registered` → `该邮箱已注册，请直接登录`
- `User not found` → `用户不存在`
- `Password should be at least 6 characters` → `密码长度至少 6 位`
- `Signups not allowed` → `当前不允许注册`
- `Rate limit exceeded` → `请求过于频繁，请稍后再试`
- `Network error`/`Fetch error` → `网络错误，请稍后重试`
- 未命中已知映射 → `发生未知错误，请稍后重试`

## 技术实现
- 在 `authErrors.js` 内做大小写不敏感的子串匹配映射，尽量覆盖 Supabase 常见信息；保留可扩展结构（对象表 + 归一化）。
- `api.js` 内统一在抛错前调用翻译函数；邀请自动邮箱确认分支若最终失败，亦走翻译。
- `App.jsx` 的登录/注册 `catch` 使用翻译函数，保证 UI 文案统一中文。

## 验证方式
- Mock 模式：用 `demo/demo`、错误密码等验证登录失败提示中文。
- Supabase 模式：
  - 使用错误密码触发 `Invalid login credentials` 映射。
  - 用未验证邮箱触发 `Email not confirmed` 映射。
  - 重复邮箱注册触发 `already registered` 映射。
- 人工检查：表单错误区与 `Toast` 均显示中文。

## 兼容与回退
- 不引入 i18n 库，仅局部中文化，风险低。
- 未识别的英文报错统一回退到中文的通用提示，避免英文直出。
