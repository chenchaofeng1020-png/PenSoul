# 竞品分析内容信息流功能设计

> 目标：在竞品详情页右侧提供一个“内容信息流”，团队成员可以针对该竞品添加不同类型的分析，并对分析进行评论。保持现有前端结构与样式体系不变（React + Tailwind，组件布局和交互框架沿用）。

## 一、范围与不改动项
- 不改动项：
  - 保持现有页面结构与主框架样式（包括 Sidebar、顶部工具栏、详情页整体布局）。
  - 继续复用现有组件与样式约定，例如 <mcfile name="RichTextEditor.jsx" path="/Users/code/product_duck/src/components/RichTextEditor.jsx"></mcfile>、<mcfile name="CommentSystem.jsx" path="/Users/code/product_duck/src/components/CommentSystem.jsx"></mcfile>、<mcfile name="NewCompetitorDetailPage.jsx" path="/Users/code/product_duck/src/components/NewCompetitorDetailPage.jsx"></mcfile>。
- 新增项：
  - 信息流列表（右侧主内容区），呈现“分析卡片”列表，支持展开查看详情。
  - 添加分析入口（浮层/抽屉/弹窗形式，具体沿用当前项目的模态/抽屉风格）。
  - 针对分析的评论区（嵌入在分析卡片底部）。

## 二、信息流结构与交互
- 信息流列表：
  - 默认按创建时间倒序展示分析卡片。
  - 支持筛选（类型、作者、标签）、关键字搜索（标题/正文）、分页或无限滚动（视项目现有分页方案）。
  - 每条分析卡片支持：展开/折叠正文、查看附件、编辑、删除、分享、下载（沿用已有交互风格）。
- 卡片通用结构（各类型一致的基础骨架）：
  - 顶部：类型标签 + 标题 + 作者头像/昵称 + 创建/更新时间 + 标签（chips）。
  - 内容区：摘要（前几行富文本预览）+ 展开后显示完整富文本内容。
  - 底部：操作按钮（评论、编辑、删除、分享、下载）与评论列表入口。

## 三、分析类型与字段
- 通用字段（所有类型共享）：
  - 标题、类型（功能分析/调研报告/产品文档/系统截图）、所属竞品ID、作者、创建时间、更新时间、标签（多选）、正文（富文本）、可见性/权限（公开/团队/仅作者）、关联链接/文档。
- 功能分析（type=feature_analysis）：
  - 功能模块/领域、评估维度（可用性/易用性/性能/安全等可多选）、优势与劣势、评分（1-5）、结论与建议。
  - 卡片样式差异：显示评分星级与维度标签，主色调建议偏蓝。
- 调研报告（type=research_report）：
  - 调研范围、方法（访谈/问卷/竞品试用等）、样本量与对象、重点发现、结论与建议、附件（PDF/链接）。
  - 卡片样式差异：突出“概要/重点发现”区，附件入口明显，主色调建议偏紫。
- 产品文档（type=product_doc）：
  - 文档类型（PRD/需求说明/设计说明等）、版本号、关联任务/问题、附件（文档/链接）。
  - 卡片样式差异：显示版本号与文档类型徽章，主色调建议偏绿。
- 系统截图（type=screenshots）：
  - 截图列表（1-N）、截图说明（可为每张图单独标注）、来源URL、采集时间、关联页面/模块。
  - 卡片样式差异：图片网格预览 + 标注说明，主色调建议偏橙。

## 四、添加分析页面（表单）设计
- 入口：在信息流列表顶部或右上角提供“添加分析”按钮，点击弹出模态/抽屉。
- 步骤：
  1) 选择类型（单选，四种之一）。
  2) 填写基础字段（通用字段）。
  3) 填写类型特有字段（随类型变化）。
  4) 富文本编辑正文（复用 <mcfile name="RichTextEditor.jsx" path="/Users/code/product_duck/src/components/RichTextEditor.jsx"></mcfile>）。
  5) 附件上传/链接添加（与现有上传方案保持一致）。
  6) 提交并返回信息流，自动插入新卡片并滚动到该项。
- 校验：必填字段校验、附件类型/大小校验、图片数量限制（系统截图）。

## 五、评论系统设计
- 评论入口：卡片底部显示评论计数与展开按钮，展开后显示评论列表与输入框。
- 功能：添加评论、编辑评论（作者本人）、删除评论（作者或管理员）、@提及（可选）、表情/Markdown（取决于现有组件能力）。
- 结构：支持按时间排序；如需楼中楼，可支持一级回复（简化实现）。
- 复用：优先复用或改造 <mcfile name="CommentSystem.jsx" path="/Users/code/product_duck/src/components/CommentSystem.jsx"></mcfile> 的交互样式与逻辑。

## 六、权限与可见性
- 可见性设置：公开（团队可见）、私有（仅作者可见）、指定成员/角色可见（可选扩展）。
- 操作权限：
  - 编辑/删除：作者本人或管理员。
  - 评论：团队成员可评论；私有分析仅作者可见和可评论。

## 七、数据模型与接口建议（后端）
- 数据模型（建议）：
  - analyses（分析）表：id, competitor_id, type, title, author_id, content_html, tags, visibility, created_at, updated_at, extra_json（存放类型特有字段，如评分、方法等）。
  - analysis_attachments（附件）表：id, analysis_id, type(file/link), url/path, meta_json。
  - comments（评论）表：id, analysis_id, author_id, content, created_at, updated_at, visibility。
- API（REST示例）：
  - GET /api/competitors/{id}/analyses?type=&author=&tag=&q=&page=
  - POST /api/competitors/{id}/analyses
  - PATCH /api/analyses/{analysisId}
  - DELETE /api/analyses/{analysisId}
  - GET /api/analyses/{analysisId}/comments
  - POST /api/analyses/{analysisId}/comments
  - PATCH /api/comments/{commentId}
  - DELETE /api/comments/{commentId}

- 编辑器：复用 <mcfile name="RichTextEditor.jsx" path="/Users/code/product_duck/src/components/RichTextEditor.jsx"></mcfile> 作为正文编辑器；根据类型动态显示额外字段输入控件。
- 评论：复用或轻量改造 <mcfile name="CommentSystem.jsx" path="/Users/code/product_duck/src/components/CommentSystem.jsx"></mcfile>，嵌入每条分析卡片底部。
- 交互：保持与现有“分享/下载/删除”等按钮交互一致，避免改变整体风格。

## 九、验收标准
- 信息流能正确按时间展示分析，并可根据类型/作者/标签筛选。
- 可添加四种类型的分析，字段校验正常，提交后立即可见。
- 富文本编辑与附件/截图上传正常，预览展示符合类型样式差异。
- 评论区可展开、添加、编辑、删除；权限与可见性策略生效。
- 操作按钮（编辑/删除/分享/下载）交互一致、反馈明确。

## 十、迭代里程碑（建议）
1) v1 基础版：信息流列表 + 添加分析（通用字段+富文本）+ 评论基础能力。
2) v1.1 类型差异样式与字段：功能分析评分、调研报告附件、产品文档版本、系统截图网格预览。
3) v1.2 筛选与搜索、分页/无限滚动、分享/下载完善。
4) v1.3 权限/可见性增强、@提及与通知（选做）。

## 十一、非功能性要求
- 性能：列表分页或懒加载；图片懒加载与压缩；避免一次性加载超大富文本。
- 安全：权限校验、附件安全（防XSS）、上传限制（类型/大小）。
- 可用性：移动端适配；可访问性（语义标签与可见标签）。
- 国际化（可选）：类型与标签支持多语言字典。

以上方案在不改变现有前端结构与样式框架的前提下，新增并细化了右侧“竞品分析信息流”的完整功能设计，覆盖类型差异、表单与评论、权限与接口、验收与迭代等要点。
