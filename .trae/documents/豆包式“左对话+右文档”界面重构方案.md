# 豆包式“左对话+右文档”内容创作界面改造方案

根据您的需求，我们将把现有的“表单+侧边栏”模式重构为“沉浸式双栏协作”模式。

## 1. 核心布局重构 (Layout Redesign)
修改 `ContentPlanningPage.jsx` 的编辑模式 (`isComposeOpen`) 布局：
- **移除**：原有的三栏布局 (`grid-cols-3`)。
- **新增**：响应式双栏布局 (Split View)。
  - **左栏 (40% - AI Copilot)**：智能助手对话区。负责接收指令、展示思考过程、提供操作建议。
  - **右栏 (60% - Document)**：沉浸式文档编辑区。负责展示和编辑内容，实时响应AI的生成结果。

## 2. 新增组件：AI 智能助手 (AiCopilotPanel)
我们将创建一个全新的组件 `AiCopilotPanel` 来替代 `AiDraftAssistant` 的侧边栏 UI，功能包括：
- **对话式交互**：
  - 用户不再填写繁琐表单，而是直接在输入框发送指令（如“帮我写一篇关于AI的大纲”）。
  - 支持快捷指令气泡（生成初稿、润色、扩写、校对）。
- **思考过程展示**：
  - 在对话流中展示 AI 的状态（“正在分析风格...”、“正在搜索素材...”、“正在撰写...”）。
  - 模拟“思考链”效果，提升用户耐心。
- **上下文感知**：
  - 能够读取右侧编辑器的当前内容作为上下文（Context）。

## 3. 交互流程设计
1. **生成模式**：
   - 用户在左侧输入主题 -> AI 在左侧确认 -> AI 调用接口 -> **结果直接流式写入右侧编辑器** -> 左侧提示“生成完毕”。
2. **修改模式**：
   - 用户在左侧输入“把第二段改得幽默一点” -> AI 读取右侧内容 -> 调用润色接口 -> **更新右侧编辑器对应内容**。
3. **文风复刻**：
   - 将原有的“文风复刻”配置折叠进左侧的“设置/模型”菜单中，保持界面整洁。

## 4. 技术实现步骤
1. **创建 `AiCopilotPanel.jsx`**：实现聊天气泡 UI、输入框、消息流状态管理。
2. **改造 `ContentPlanningPage.jsx`**：
   - 引入 `AiCopilotPanel`。
   - 调整 CSS 布局为 Flex/Grid 双栏。
   - 建立父组件状态桥接，使左侧 Chat 能控制右侧 Editor 的内容 (`composeBody`)。
3. **逻辑迁移**：
   - 将 `AiDraftAssistant` 中的 `handleGenerate`, `handleRefine` 等核心逻辑迁移至 `AiCopilotPanel`，适配对话触发方式。

该方案将完全改变内容创作的体验，使其更符合当前主流 AI 编辑器（如 Notion AI、豆包、Cursor）的交互习惯。
