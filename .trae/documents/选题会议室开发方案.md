# 开发方案：选题会议室 (Ideation Conference Room)

## 1. 概述
本模块旨在实现“选题会议室”功能，模拟与 AI Agent（诸葛）的战略会议，生成选题策划案。模块采用左中右三栏布局，分别对应会话管理、对话交互和策略看板。

## 2. 数据库设计 (Supabase)

需要创建以下新表来支持功能：

### 2.1 `ideation_sessions` (会话表)
存储每一次选题会议的基础信息。
- `id`: UUID, Primary Key
- `user_id`: UUID, Foreign Key (auth.users)
- `product_id`: UUID, Foreign Key (products) - *可选，如果需要关联产品*
- `persona_id`: String, 当前选用的人设 ID (如 'p_tech_01')
- `title`: String, 会话标题 (自动生成或手动修改)
- `created_at`: Timestamp
- `updated_at`: Timestamp

### 2.2 `ideation_messages` (消息表)
存储对话记录。
- `id`: UUID, Primary Key
- `session_id`: UUID, Foreign Key (ideation_sessions)
- `role`: String ('user' | 'assistant' | 'system')
- `content`: Text, 消息内容
- `created_at`: Timestamp

### 2.3 `ideation_topics` (选题表)
存储生成的选题卡片。
- `id`: UUID, Primary Key
- `session_id`: UUID, Foreign Key (ideation_sessions)
- `title`: String, 选题标题
- `angle`: String, 角度标签 (如 '情绪', '商业', '反转')
- `hook`: String, 核心钩子
- `status`: String ('pending' | 'scheduled' | 'published')
- `schedule_date`: Date, 排期日期
- `detail_json`: JSONB, 完整详情
- `created_at`: Timestamp

## 3. 前端架构设计

### 3.1 路由与入口
- **路由**: 在 `App.jsx` 中处理 `normalizedSelectedCategory === '选题会议室'`。
- **入口**: 在 `Sidebar.jsx` 的 `menuItems` 中添加：
  ```javascript
  { icon: '💡', label: '选题会议室', path: '/ideation' }
  ```
  *(注：原“智能选题”可能需要保留或替换，根据具体需求定，此处暂定为新增)*

### 3.2 组件结构
新建目录 `src/components/IdeationConference/`：

- **`index.jsx`**: 主容器组件，负责布局管理 (Grid 布局)。
- **`SessionNavigator.jsx`**: 左侧栏
  - 显示历史会话列表。
  - “新建策划”按钮。
  - 人设选择下拉框。
- **`ChatInterface.jsx`**: 中间栏
  - 消息列表渲染。
  - 输入框组件。
  - 负责调用 LLM API (`/api/agents/test`)。
- **`StrategyBoard.jsx`**: 右侧栏
  - 选题卡片流 (`TopicCard` 列表)。
  - 负责卡片的展示和状态更新（如加入排期）。
- **`TopicCard.jsx`**: 单个选题卡片组件。

### 3.3 状态管理
使用 `IdeationContext` 或在 `index.jsx` 中提升状态：
- `currentSessionId`: 当前激活的会话 ID。
- `persona`: 当前选择的人设。
- `messages`: 当前会话的消息列表。
- `topics`: 当前会话生成的选题列表。

## 4. 核心逻辑实现

### 4.1 会话管理
- **新建**: 点击按钮 -> 插入 `ideation_sessions` -> 更新 `currentSessionId` -> 触发 Agent 开场白。
- **切换**: 点击左侧列表 -> 获取对应 session 的 messages 和 topics -> 更新 UI。
- **自动命名**: 在第 2-3 轮对话后，后台调用 LLM 总结标题并更新 `ideation_sessions.title`。

### 4.2 对话交互
- 用户发送消息 -> 插入 `ideation_messages` (user) -> UI 立即显示。
- 调用 `/api/agents/test` 接口：
  - `system_prompt`: 诸葛的人设 Prompt。
  - `user_message`: 用户输入 + 历史上下文 (最近 N 条)。
- 接收回复 -> 插入 `ideation_messages` (assistant) -> UI 显示。

### 4.3 选题生成 (Tool Calling 模拟)
- 诸葛的 Prompt 中会包含指令：当需要生成选题时，输出特定格式的 JSON 或分隔符。
- 前端解析 Agent 回复：
  - 如果检测到 JSON 格式的选题数据 -> 自动插入 `ideation_topics` 表 -> 右侧看板自动刷新。
  - 过滤掉 JSON 部分，只将自然语言回复显示在对话框中（或保留提示“已生成方案”）。

### 4.4 排期联动
- 点击卡片上的“加入排期” -> 弹出日期选择 -> 更新 `ideation_topics.status` 和 `schedule_date`。
- *(可选)* 同步写入到全局的内容日历表中（如果存在）。
