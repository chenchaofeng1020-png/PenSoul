# 团队成员管理功能设计方案

## 1. 项目概述

### 1.1 功能背景
当前产品鸭系统中，新建项目时需要手动输入团队成员信息，缺乏统一的团队成员管理功能。为了提升用户体验和团队协作效率，需要在设置板块开发团队成员管理相关功能，实现团队成员的统一管理和项目成员的快速选择。

### 1.2 核心目标
- 提供统一的团队成员管理界面
- 支持团队成员的增删改查操作
- 实现项目创建时的成员快速选择
- 建立完善的权限管理体系
- **确保每个产品的团队独立管理，做好数据隔离**
- 提升团队协作效率

### 1.3 功能范围
- 团队成员管理（增删改查）
- 成员邀请功能（基于分享链接）
- 权限角色管理
- 项目成员选择器
- 成员状态管理
- **产品级别的团队隔离**

## 2. 产品设计

### 2.0 团队隔离原则

#### 2.0.1 产品级别隔离
- **每个产品拥有独立的团队**：用户在不同产品中可以拥有不同的角色和权限
- **数据完全隔离**：产品A的团队成员无法访问产品B的任何数据
- **权限独立管理**：用户在产品A是Owner，在产品B可能是Member或无权限
- **邀请链接产品绑定**：每个邀请链接只能加入特定产品的团队

#### 2.0.2 隔离实现要点
- 所有API接口必须包含产品ID参数进行权限验证
- 前端组件需要基于当前产品上下文显示数据
- 数据库查询必须添加产品ID过滤条件
- 邀请链接包含产品信息，确保用户加入正确的团队

### 2.1 用户角色定义

#### 2.1.1 角色类型
- **Owner（所有者）**: 产品创建者，拥有所有权限
- **Admin（管理员）**: 可管理团队成员和项目设置
- **Member（成员）**: 可参与项目协作，查看和编辑内容
- **Viewer（观察者）**: 只读权限，可查看项目内容

#### 2.1.2 权限矩阵
| 功能 | Owner | Admin | Member | Viewer |
|------|-------|-------|--------|--------|
| 邀请成员 | ✅ | ✅ | ❌ | ❌ |
| 移除成员 | ✅ | ✅ | ❌ | ❌ |
| 修改成员角色 | ✅ | ✅ | ❌ | ❌ |
| 创建项目 | ✅ | ✅ | ✅ | ❌ |
| 编辑项目 | ✅ | ✅ | ✅ | ❌ |
| 查看项目 | ✅ | ✅ | ✅ | ✅ |
| 删除项目 | ✅ | ✅ | ❌ | ❌ |

### 2.2 用户流程设计

#### 2.2.1 团队成员管理流程
1. 用户进入设置页面
2. 选择"团队成员"选项卡
3. 查看当前团队成员列表
4. 执行成员管理操作（邀请、编辑、删除）

#### 2.2.2 成员邀请流程
1. 点击"邀请成员"按钮
2. 选择成员角色并生成邀请链接
3. 复制分享链接发送给被邀请者
4. 被邀请者通过链接访问注册/登录页面
5. 自动加入团队

#### 2.2.3 项目创建成员选择流程
1. 在新建项目时，显示成员选择器
2. 从团队成员列表中选择项目成员
3. 为每个成员分配项目角色
4. 完成项目创建

```sql
ALTER TABLE user_products ADD COLUMN (

CREATE TABLE member_invitations (
    id INT PRIMARY KEY AUTO_INCREMENT,
```

```php
<?php

namespace App\Controllers;

use App\Models\User;
use App\Models\Product;
use App\Models\UserProduct;
use App\Models\MemberInvitation;
use App\Utils\Response;
use App\Utils\EmailService;

class TeamMemberController extends BaseController
{
    // 获取团队成员列表
    public function getMembers($productId)
    {
        // 权限检查
        if (!$this->hasProductAccess($productId, ['owner', 'admin', 'member'])) {
            Response::error('无权限访问', 403);
            return;
        }

        $userProduct = new UserProduct();
        $members = $userProduct->getProductMembers($productId, $_GET);
        
        Response::success($members);
    }

    // 邀请成员
    public function inviteMember($productId)
    {
        // 权限检查
        if (!$this->hasProductAccess($productId, ['owner', 'admin'])) {
            Response::error('无权限邀请成员', 403);
            return;
        }

        $data = $this->getRequestData();
        
        // 验证角色有效性
        $validRoles = ['owner', 'admin', 'member', 'viewer'];
        if (!in_array($data['role'], $validRoles)) {
            Response::error('无效的角色类型', 400);
            return;
        }

        // 创建邀请记录
        $invitation = new MemberInvitation();
        $invitationData = [
            'product_id' => $productId,
            'role' => $data['role'] ?? 'member',
            'invited_by' => $this->user['id'],
            'invitation_token' => bin2hex(random_bytes(32)),
            'expires_at' => date('Y-m-d H:i:s', strtotime('+7 days'))
        ];

        $invitationId = $invitation->create($invitationData);
        
        // 生成邀请链接
        $inviteUrl = "https://app.com/invite/{$invitationData['invitation_token']}";

        Response::success([
            'invitation_id' => $invitationId,
            'invitation_token' => $invitationData['invitation_token'],
            'invite_url' => $inviteUrl,
            'role' => $data['role'],
            'expires_at' => $invitationData['expires_at']
        ], '邀请链接已生成');
    }

    }

        // 权限检查和实现
```

### 5.4 邀请处理服务

#### 5.4.1 邀请链接处理
```php
<?php

namespace App\Controllers;

class InvitationController extends BaseController
{
    // 获取邀请信息
    public function getInvitation($token)
    {
        $invitation = new MemberInvitation();
        $inviteData = $invitation->getByToken($token);
        
        if (!$inviteData || $inviteData['expires_at'] < date('Y-m-d H:i:s')) {
            Response::error('邀请链接无效或已过期', 404);
            return;
        }

        $product = new Product();
        $productInfo = $product->getById($inviteData['product_id']);
        
        $inviter = new User();
        $inviterInfo = $inviter->getById($inviteData['invited_by']);

        Response::success([
            'product' => [
                'id' => $productInfo['id'],
                'name' => $productInfo['name']
            ],
            'role' => $inviteData['role'],
            'invited_by' => $inviterInfo['name'],
            'expires_at' => $inviteData['expires_at']
        ]);
    }

    // 接受邀请
    public function acceptInvitation($token)
    {
        $invitation = new MemberInvitation();
        $inviteData = $invitation->getByToken($token);
        
        if (!$inviteData || $inviteData['expires_at'] < date('Y-m-d H:i:s')) {
            Response::error('邀请链接无效或已过期', 404);
            return;
        }

        // 检查用户是否已是成员
        $userProduct = new UserProduct();
        if ($userProduct->isMember($inviteData['product_id'], $this->user['id'])) {
            Response::error('您已是该产品的团队成员', 400);
            return;
        }

        // 添加用户到产品团队
        $userProduct->addMember([
            'user_id' => $this->user['id'],
            'product_id' => $inviteData['product_id'],
            'role' => $inviteData['role'],
            'invited_by' => $inviteData['invited_by'],
            'joined_at' => date('Y-m-d H:i:s')
        ]);

        // 标记邀请为已使用
        $invitation->markAsUsed($token);

        $product = new Product();
        $productInfo = $product->getById($inviteData['product_id']);

        Response::success([
            'product' => [
                'id' => $productInfo['id'],
                'name' => $productInfo['name']
            ],
            'role' => $inviteData['role']
        ], '已成功加入团队');
    }

    // 拒绝邀请
    public function declineInvitation($token)
    {
        $invitation = new MemberInvitation();
        $invitation->markAsDeclined($token);
        
        Response::success(null, '已拒绝邀请');
    }
}
```

## 12. 问题和不确定性

### 12.1 需要确认的问题

1. **邀请机制**：
   - 是否支持批量邀请多个成员？
   - 邀请链接的有效期设置（当前设计为7天）？
   - 邀请链接是否可以重复使用？

2. **权限粒度**：
   - 是否需要更细粒度的权限控制？
   - 不同模块（竞品分析、路线图等）是否需要独立权限？

3. **成员上限**：
   - 每个产品的成员数量是否有限制？
   - 不同版本是否有不同的成员限制？

4. **数据迁移**：
   - 现有产品如何处理成员关系？
   - 是否需要将现有的owner自动设置为团队成员？

5. **通知机制**：
   - 除了分享链接，是否需要站内通知？
   - 成员状态变更是否需要通知其他成员？

6. **集成范围**：
   - 除了项目创建，还有哪些地方需要成员选择？
   - 评论系统是否需要@成员功能？

### 12.2 技术选型确认

1. **链接安全性**：邀请链接的安全措施和防滥用机制？
2. **文件存储**：成员头像存储方案？
3. **缓存策略**：成员信息是否需要缓存？
4. **实时通信**：是否需要WebSocket支持实时通知？

这些问题需要在开发前与产品团队确认，以确保功能设计符合实际需求和技术约束。
